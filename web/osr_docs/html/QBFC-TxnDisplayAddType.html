TxnDisplayAddType
Specifies the type of QuickBooks transaction for which to open an edit form in the QuickBooks UI.

Notice that some transaction types do not support the entity reference that "prefills" the form because it would make no sense to do this for certain forms. Accordingly, BillPayment does not support the entity reference, nor does Deposit, Charge, SalesTaxPaymentCheck, JournalEntry, or InventoryAdjustment. If you add an entity reference for those elements that don't support it, you will get a "conflicting parameter" error at runtime.
