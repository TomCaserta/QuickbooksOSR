Specifies the type of QuickBooks transaction for which to open an "add" form in the QuickBooks UI. Notice that some of the QuickBooks transaction types do not support the entity reference that "prefills" the form because it would make no sense to do this for certain forms. Accordingly, BillPayment does not support the entity reference, nor does Deposit, Charge, SalesTaxPaymentCheck, JournalEntry, or InventoryAdjustment.

If you add an entity reference for those elements that don't support it, you will get a "conflicting parameter" error at runtime.

Values:
Bill, BillPayment, BuildAssembly, Charge, Check, CreditCardCharge, CreditCardCredit, CreditMemo, Deposit, Estimate, InventoryAdjustment, Invoice, ItemReceipt, JournalEntry, PurchaseOrder, ReceivePayment, SalesOrder, SalesReceipt, SalesTaxPaymentCheck, VendorCredit

No default value.
