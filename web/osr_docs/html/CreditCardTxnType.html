CreditCardTxnType
Used in a Sales Receipt, Receive Payment, or ARRefundCreditCard request to indicate the QBMS transaction from which the data in this CreditCardTxnInfo block originated, for example, from a QBMS Charge, Authorization, Capture, Refund, or VoiceAuthorization. One main use of this is for QuickBooks to recognize a ReceivePaymentAdd to know whether the transaction is an Auth, in which case the ReceivePayment is non posting, or a Charge, in which case the ReceivePayment is posted.

The CreditCardTxnType that can be used for each supported transaction is enforced by the business logic layer, not by the qbXML spec. The logic imposes the following limitations:

ARRefundCreditCardAdd allows only Refund.
ReceivePaymentAdd/Mod allows Authorization, Capture, Charge, and VoiceAuthorization.
SalesReceiptAdd allows types Charge, Refund, and VoiceAuthorization ( Refund is allowed because line items can represent a Refund)

This field is optional, but if not supplied, the default for the transaction will be used. For example, in ARRefundCreditCard transactions, the default is Refund, for SalesReceipt the default is Charge.
